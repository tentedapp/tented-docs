## User Management Overview

Managing users in your Tented workspace involves overseeing user accounts, updating permissions, monitoring activity, and ensuring proper access control. This guide covers all aspects of user management.

## User Dashboard

### User List Overview

**User Information Display:**
- **Name**: User's display name
- **Email**: User's email address
- **Role**: Current role assignment
- **Status**: Active, inactive, or pending
- **Last Active**: When user last accessed workspace
- **Join Date**: When user joined workspace

**Quick Actions:**
- **Edit User**: Modify user details
- **Change Role**: Update user permissions
- **Deactivate**: Temporarily disable access
- **Remove**: Permanently remove user

### User Status Indicators

**Active Users:**
- Currently have access to workspace
- Can perform actions based on role
- Show recent activity
- Green status indicator

**Inactive Users:**
- Temporarily disabled access
- Cannot log in or perform actions
- Can be reactivated
- Gray status indicator

**Pending Users:**
- Invited but haven't accepted
- Awaiting invitation response
- Can resend or cancel invitation
- Yellow status indicator

## Role Management

### Changing User Roles

**Role Update Process:**
1. Select user from user list
2. Click "Change Role" or "Edit"
3. Select new role from dropdown
4. Confirm role change
5. User receives notification

**Role Change Effects:**
- **Immediate**: Changes take effect immediately
- **Permissions**: User access updates instantly
- **Notifications**: User notified of changes
- **Audit Trail**: Change logged for compliance

### Role Hierarchy

**Admin Role:**
- Full workspace access
- User management capabilities
- Billing and settings access
- Can assign all roles

**Editor Role:**
- Content creation and editing
- Publishing capabilities
- Data access and export
- Cannot manage users

**Viewer Role:**
- Read-only access
- Data viewing and export
- Limited functionality
- Cannot make changes

**Custom Roles:**
- Specific permission sets
- Tailored access levels
- Custom functionality
- Flexible permissions

## User Deactivation and Removal

### Deactivating Users

**Temporary Deactivation:**
- Disable user access temporarily
- Preserve user data and history
- Can be reactivated later
- Maintains audit trail

**Deactivation Reasons:**
- **Temporary Leave**: User on leave
- **Security Concerns**: Suspicious activity
- **Role Changes**: Pending role updates
- **Account Issues**: Technical problems

### Removing Users

**Permanent Removal:**
- Permanently delete user account
- Remove all user data
- Cannot be undone
- Requires confirmation

**Data Handling:**
- **Data Transfer**: Transfer data to other users
- **Data Export**: Export user's data
- **Data Retention**: Follow retention policies
- **Data Deletion**: Remove all user data

**Removal Process:**
1. **Backup Data**: Export user's data
2. **Transfer Ownership**: Transfer tents to other users
3. **Notify User**: Inform user of removal
4. **Confirm Removal**: Final confirmation
5. **Complete Removal**: Delete account and data

## Troubleshooting User Management

### Common Issues

**User Access Problems:**
- Verify user role and permissions
- Check user status (active/inactive)

**Role Assignment Issues:**
- Confirm role changes were saved
- Check for conflicting permissions
- Verify user received notification
- Test user access

**Data Access Problems:**
- Review data sharing settings
- Check workspace permissions
- Verify user group memberships
- Contact administrator
