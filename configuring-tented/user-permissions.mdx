## User Permissions Overview

User permissions in Tented control what users can see, create, edit, and manage within your workspace. Understanding and properly configuring permissions is essential for maintaining security, collaboration, and workflow efficiency.

## Permission Levels

### Admin Permissions

**Full Workspace Access:**
- **User Management**: Invite, edit, and remove users
- **Billing Access**: View and manage subscription
- **Settings Control**: Modify workspace settings
- **Data Access**: Full access to all data

**System Administration:**
- **Security Settings**: Configure security options
- **Integration Management**: Set up third-party integrations
- **Backup and Recovery**: Manage data backups
- **Audit Access**: View all audit logs

**Content Management:**
- **All Tents**: Create, edit, and delete any tent
- **All Forms**: Access all form submissions
- **All Analytics**: View all performance data
- **Publishing Control**: Publish/unpublish any tent

### Editor Permissions

**Content Creation:**
- **Tent Creation**: Create new tents
- **Tent Editing**: Edit assigned tents
- **Tent Publishing**: Publish tents
- **Form Management**: Handle form submissions

**Data Access:**
- **Tent Data**: Access tent content and settings
- **Form Submissions**: View and export form data
- **Analytics**: View tent performance metrics
- **Reports**: Generate and export reports

**Collaboration:**
- **Team Communication**: Participate in team discussions
- **Content Sharing**: Share tents with team members
- **Feedback**: Provide and receive feedback
- **Version Control**: Access tent version history

### Viewer Permissions

**Read-Only Access:**
- **Tent Viewing**: View published tents
- **Data Viewing**: View tent data and analytics
- **Form Data**: View form submission data
- **Reports**: View generated reports

**Export Capabilities:**
- **Data Export**: Export tent data
- **Form Export**: Export form submissions
- **Report Export**: Export analytics reports
- **Code Export**: Download tent code

**Limited Actions:**
- **Preview Access**: Preview tents before publishing
- **Comment Access**: Add comments to tents
- **Share Access**: Share tent previews
- **Download Access**: Download tent files

## Permission Inheritance

### Role-Based Inheritance

**Default Permissions:**
- Users inherit permissions from their role
- Role changes update all permissions
- Custom permissions override role defaults
- Permission conflicts resolved by hierarchy

**Custom Overrides:**
- Override specific permissions for users
- Maintain role-based defaults
- Granular control over access

## Permission Management

### Setting Permissions

**User Permission Assignment:**
1. Navigate to user management
2. Select user to modify
3. Click "Permissions" or "Edit"
4. Configure specific permissions
5. Save permission changes

**Role Permission Assignment:**
1. Go to role management
2. Select role to modify
3. Configure role permissions
4. Apply to all users with role
5. Notify users of changes
